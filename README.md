# ⚙️ ИНДУСТРИЯ МАНИПУЛЯТОР

### Анализатор проектной документации

**Автоматическая проверка EPLAN-схем и таблиц подключений на ошибки**

Извлечение данных из PDF → детерминированный чекер → два ИИ-агента → отчёт по замечаниям.
Полностью офлайн, локальные LLM (LM Studio).

`V1.1 beta` · Python 3.10+ · без облака

</div>

---

## Что это

Инструмент находит ошибки в комплекте проектной документации: несостыковки между
**таблицей подключений (нетлистом)** и **принципиальной схемой**, а также ошибки
внутри каждого документа (дубли адресов, битые перекрёстные ссылки, неверный формат
KKS, незаполненные поля).

На выходе — таблица замечаний с точным указанием, **где** (лист, строка, шкаф, клемма,
маркировка, KKS, проводник), **что найдено** и **что требуется исправить**.

## Из чего состоит

```
industry-manipulator/
├── web_app/              ← локальный веб-интерфейс (загрузка PDF, запуск, отчёт)
│   ├── server.py         · бэкенд на стандартной библиотеке Python (без зависимостей)
│   └── static/           · страница интерфейса (HTML/CSS/JS)
│
└── analyzer_to_errors/   ← пайплайн анализа (ядро)
    ├── main.py           · оркестратор всех стадий
    ├── ingest.py         · PDF → структурированные данные
    ├── oi_agent.py       · ИИ-агенты (Open Interpreter)
    ├── merge_reports.py  · сшивание отчётов, дедупликация
    └── data/             · песочница: исходные PDF, скрипты-парсеры, извлечённые данные
```

> Подробное устройство пайплайна, формат находок и схема данных —
> в [analyzer_to_errors/README.md](analyzer_to_errors/README.md).

## Быстрый старт

```bash
# 1. Зависимости
pip install -r analyzer_to_errors/requirements.txt

# 2. Веб-интерфейс
python web_app/server.py            # → http://localhost:8000
python web_app/server.py --port 9000
```

Дальше — в браузере:

1. **Перетащите PDF** в область загрузки (принимаются только `.pdf`).
2. **Укажите тип** каждого документа — *принципиальная схема* или *нетлист*.
3. **Запустите анализ**:
   - **Полный** (скрипты + ИИ) — извлечение, детерминированный чекер и два ИИ-агента. Требует запущенного сервера ИИ.
   - **Без ИИ** — только извлечение и детерминированный чекер таблиц подключений. Быстро, ИИ не нужен.
4. Смотрите **отчёт** таблицей и живую **консоль** прогона.
5. Анализ можно **отменить** в любой момент кнопкой «Отменить анализ» — пайплайн
   остановится на ближайшей границе стадии.

### Без веб-интерфейса (CLI)

```bash
cd analyzer_to_errors
python main.py --extract-only    # только парсеры PDF (LLM не нужны)
python main.py --rules-only      # только детерминированный чекер
python main.py                   # весь пайплайн → output/merged_report.json
```

## Требования

- **Python 3.10+**
- Зависимости пайплайна — `analyzer_to_errors/requirements.txt`
  (`openai`, `pyyaml`, `jsonschema`, `open-interpreter`, `pdfplumber`, `PyMuPDF`).
- Для полного анализа — локальный сервер **LM Studio** с двумя загруженными моделями.
  Адреса и модели настраиваются в `analyzer_to_errors/config.yaml` (секция `llm_servers`).
  Доступность серверов и моделей проверяется прямо в интерфейсе кнопкой
  «Проверить серверы и модели ИИ».

Веб-серверу отдельные зависимости не нужны — он на стандартной библиотеке (`http.server`).

## Как работает анализ (коротко)

| Стадия | Что делает | Нужен ИИ |
|---|---|---|
| **0. Извлечение** | PDF → JSON с данными по каждому документу (`ingest.py`) | нет |
| **1. Правила** | Детерминированный чекер таблиц подключений находит твёрдые ошибки | нет |
| **2. Агенты** | Два LLM-агента независимо анализируют данные, пишут и исполняют код | да |
| **3. Сшивание** | Объединение отчётов, удаление дублей и заранее известных ошибок | да |
| **4. Валидация** | Каждый JSON-ответ модели проверяется по схеме с авторемонтом | да |

Находки чекера — *ground truth*: они добавляются в итог детерминированно, уже после
работы моделей, и не теряются на сшивании.

## Отмена анализа

Анализ запускается отдельным процессом, поэтому отмена **мгновенная**: кнопка
«Отменить анализ» убивает этот процесс целиком вместе со всеми потомками —
эквивалент Ctrl+C во всех его окнах разом, только гарантированный (сетевой вызов
к ИИ или Open Interpreter кооперативную отмену вполне мог бы проигнорировать).
Консоль прогона при этом не очищается — в неё лишь дописывается отметка об отмене.
Частичные результаты незавершённого прогона (`output/` и рабочая папка агента)
подчищаются автоматически сразу после остановки; уже извлечённые данные документов
и файлы пользователя не трогаются.

## Настройка

Всё в `analyzer_to_errors/config.yaml`: адреса и модели LLM-серверов (`llm_servers`),
пути (`paths`), число попыток авторемонта JSON и лимиты агентов (`agent`),
`extraction.reuse_existing` — не перепарсивать PDF, если данные уже извлечены.
